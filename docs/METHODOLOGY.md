# Methodology

promptPACK uses advanced analogical engineering: when prompt-compression precedent is immature or unstable, we borrow from older, mature disciplines.

## Primary analogies

| promptPACK problem | Mature analogy | Practice borrowed |
|---|---|---|
| Token compression | File compression | Do not claim compression if output expands |
| Prompt optimization | Compiler design | Preserve semantics and report warnings |
| Objective advising | Intake / diagnostic triage | Infer likely goal, explain confidence, allow override |
| Risk scoring | Medical/legal severity routing | High-risk material gets conservative handling |
| Browser insertion | Deployment safety | User confirmation before state change |
| Preset testing | Regression testing | Known cases must keep passing |

## Core rules

1. No fake compression.
2. Preserve locked constraints and must-facts.
3. No-op is a valid result.
4. Transform objectives require transformed handoff packets.
5. High-stakes/audit content routes to human review.
6. Browser insertion requires explicit user action.
