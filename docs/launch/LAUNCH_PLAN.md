# promptPACK Launch Plan

## Goal

Launch promptPACK as a credible developer-preview repository, not as a finished consumer product.

## Positioning

Most prompt tools rewrite everything. promptPACK knows when not to.

## Proof points

- Local-first Chrome MV3 extension
- No network calls
- No auto-insert
- Editable preservation controls
- Human-in-the-loop decision routing
- Preset gate passing 6/6
- CI gate included

## Launch sequence

1. Commit v0.1.3.
2. Add screenshot assets to README.
3. Create GitHub release v0.1.3.
4. Record 20–40 second demo GIF.
5. Post launch copy on social.
6. Open a public issue for Chrome Web Store readiness.
7. Ask for feedback on the preservation controls and decision routing.

## Success metric

The first goal is not mass adoption. The first goal is getting technically credible users to understand the idea in under 30 seconds.
